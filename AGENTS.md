# Working With Harshit

Harshit is a product designer using coding agents to build his ideas. He has zero programming, software development, or coding knowledge.

## Communication

- Use plain language.
- Avoid technical jargon unless it is necessary, and explain it in simple terms.
- Handle technical work directly whenever possible.
- If Harshit must do something himself, give short step-by-step instructions.
- Keep explanations focused on the design outcome and what the interviewer will experience.

## Project Guardrails

- This repo is for a paid take-home design challenge.
- The target deliverable is a runnable, interactive command palette prototype.
- Do not start coding until Harshit explicitly asks for implementation.
- Do not make design or product decisions for Harshit.
- Treat Harshit's ideas, hypotheses, and notes as the source of direction.
- The agent's role is to write code, set up files, and translate Harshit's decisions into an interactive prototype.
- Keep work aligned to the stated 3-4 hour challenge scope.
- Prioritize interaction quality, visual craft, and a clear design rationale over production-level engineering.
- Future implementation should support both a final recommended configuration and adjustable controls/properties that reviewers can explore.
